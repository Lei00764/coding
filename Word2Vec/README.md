# 如何表示一个词

要求：

1. 因为要利用计算机来处理，语义关系应该是数字化的
2. 这个数值要能够体现词与词之间的语义关系

## 分词器/标记器

概念：把所有 token 投射到一个一维空间，每个 token 用一个唯一的整数表示。

缺点：无法体现词与词之间的语义关系。

例子：用 1 表示 `apple`，用 2 表示 `banana`，用 3 表示 `cherry`。

## One-hot encoding (独热编码)

概念：为每一个 token 分配一个单独的维度，该维度的值为 1，其余维度的值为 0。

缺点：向量的维度很大，信息密度过低；向量之间的相似度无法体现词之间的语义关系。

例子：`apple` 的 one-hot encoding 为 `[1, 0, 0]`，`banana` 的 one-hot encoding 为 `[0, 1, 0]`，`cherry` 的 one-hot encoding 为 `[0, 0, 1]`。

## Word2Vec

概念：Word2Vec 是一种词嵌入技术，它将每个 token 投射到一个低维空间（降维），使得词与词之间的语义关系能够被体现。

Word2Vec 有两种模型：CBOW 和 Skip-gram。

CBOW：根据上下文预测中心词。

Skip-gram：根据中心词预测上下文。

Word2Vec 的输出是一个词向量矩阵，每一行是一个 token 的词向量。

重点是得到一个词向量矩阵，要能体现词与词之间的相互关系，至于预测结果，是否准确不太重要。
